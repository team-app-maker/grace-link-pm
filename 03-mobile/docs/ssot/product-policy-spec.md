# GraceLink 제품 정책 문서

> SSOT 연계 문서: `docs/SSOT.md`

## 1. 문서 목적

이 문서는 GraceLink 현재 제품의 정책을 한 문서에 모아 둔 **정책 SSOT**다.

이 문서를 읽는 목적은 다음과 같다.
- 제품 규칙을 UI 설명과 분리해서 본다.
- 비용/타이머/검증/운영 정책을 한 번에 확인한다.
- 기능을 바꿀 때 어떤 정책이 깨지는지 즉시 알 수 있게 한다.

## 2. 정책 우선순위

정책 판단은 아래 순서로 따른다.
1. 현재 앱 코드/`src/mock/*`
2. `docs/api/*`
3. 기존 상위 명세

즉, 이 문서는 **현재 구현된 정책의 정리본**이다.

## 3. 사용자 상태 정책

| 상태 | 의미 | 진입 가능 |
| --- | --- | --- |
| `SUBMITTED` | 로그인/제출 전 또는 제출 직후 초기 상태 | 온보딩 |
| `UNDER_REVIEW` | 심사 대기 | 심사 대기 화면 |
| `APPROVED` | 승인 완료 | 탭/메인 기능 |
| `REJECTED` | 반려 | 현재 메인 UX 미정 |
| `NEEDS_EDIT` | 수정 필요 | 현재 메인 UX 미정 |
| `SUSPENDED` | 정지 | 로그인 제한 대상 |

### 핵심 규칙
- 승인 전에는 메인 추천 경험 진입 불가
- `UNDER_REVIEW`는 심사 대기 화면으로 강제 이동
- `APPROVED`만 추천/매칭/광고 보상 핵심 기능 사용 가능

## 4. 온보딩 정책

### 4.1 단계 구조
- STEP 1 PASS 인증
- STEP 2 기본 정보
- STEP 3 교회 등록
- STEP 4 태그
- STEP 5 말씀/찬양
- STEP 6 신앙 질문
- STEP 7 제출 전 확인

### 4.2 검증 규칙
| 항목 | 정책 |
| --- | --- |
| 닉네임 | 2~10자 |
| 성별 | 필수 |
| 출생연도 | 4자리 숫자, 만 19세 이상 |
| 태그 | 5~15개 |
| 말씀 | 최소 1개 |
| L1 질문 | 정확히 3개 |
| L1 답변 | 각 150자 이상 |

### 4.3 교회 등록
- 검색 결과에서 선택 가능
- 없으면 등록 요청 가능
- 선택 또는 요청이 완료되어야 다음 단계 진행 가능

### 4.4 제출
- 제출 후 `UNDER_REVIEW` 전환
- 심사 완료 전 메인 추천 진입 금지

## 5. 추천/카드 정책

### 5.1 기본 추천
- 기본 추천 수: 2명
- 추가 추천: +2명
- 추가 추천 비용: 만나 1

### 5.2 카드 상태
| 상태 | 의미 |
| --- | --- |
| `ACTIVE` | 기본 노출 상태 |
| `LOCKED` | 시간 경과 후 잠금 |
| `SEEN` | 열람 완료 |
| `PASSED` | 넘김 처리 |
| `DELETED` | 삭제 만료 |

### 5.3 상태 전이
- `ACTIVE` + 48시간 경과 + 미열람 → `LOCKED`
- 생성 후 7일 경과 → `DELETED`
- pass 시 → `PASSED`

### 5.4 카드 행동 비용
| 행동 | 비용 |
| --- | --- |
| 추가 추천 +2 | 만나 1 |
| LOCKED 카드 열기 | 만나 1 |
| 보관 연장 | 만나 1 |
| 삭제 카드 복구 | 만나 2 |

## 6. 보관함 정책

### 6.1 보관 연장
- 3일 연장
- 프로필당 최대 2회
- 비용: 만나 1

### 6.2 삭제 카드 복구
- 주 2회 제한
- 비용: 만나 2

### 6.3 공통 규칙
- 잔액 부족 시 paywall 진입
- 결제/광고 완료 후 원래 액션 재실행

## 7. 매칭 정책

### 7.1 비용
| 행동 | 비용 |
| --- | --- |
| NORMAL 신청 | 만나 3 |
| SUPER 신청 | 만나 6 |
| NORMAL 수락 | 만나 3 |
| SUPER 수락 | 무료 |

### 7.2 응답 시간
- 인박스 응답 마감: 48시간

### 7.3 환급
- 거절 시 신청자 전액 환급
- 48시간 무응답 만료 시 신청자 전액 환급

### 7.4 페널티
- 무응답/미응답은 exposure weight 페널티 반영
- 본 경우와 안 본 경우 factor 다름
  - seen: 0.7
  - unseen: 0.9

## 8. 채팅 정책

### 8.1 생성
- `ACCEPTED` 매칭 시 채팅방 생성

### 8.2 마감
| 규칙 | 시간 |
| --- | --- |
| 첫 메시지 마감 | 24시간 |
| 마지막 meaningful 메시지 이후 답장 마감 | 24시간 |

### 8.3 종료
- 첫 메시지 미발송 24시간 경과 시 자동 종료
- 의미 있는 마지막 메시지 이후 24시간 무응답 시 종료

### 8.4 의미 있는 메시지 규칙
- trim 후 5자 이상
- 한글/영문 포함
- 순수 `ㅋㅎㅠㅜ` 반복만으로 이루어진 메시지 불가

### 8.5 비대칭 보상
- 상대 무응답 시 선톡자 보너스 만나 +1 가능
- 주 3회 상한
- 동일 상대 페어 1회 제한

## 9. 만나 / 결제 / 광고 정책

### 9.1 재화 구조
- `paid_manna_balance`
- `bonus_manna_balance`
- 소비 우선순위: bonus → paid

### 9.2 스토어
- 기본 패키지 예시: `manna_10`, `₩2,900`
- 구매 성공 시 paid 잔액 증가

### 9.3 광고 보상
| 항목 | 현재 정책 |
| --- | --- |
| 1회 보상 | +1 bonus 만나 |
| 일일 상한 | 3회 |
| 쿨다운 | 10분 |
| placement | `paywall`, `my_free_manna`, `empty_state` |

### 9.4 광고 보상 조건
- `APPROVED` 사용자만 가능
- 토큰 재사용 불가
- 일일 카운터는 KST 기준 리셋

### 9.5 Paywall 정책
- `INSUFFICIENT_MANNA` 발생 시 하단 paywall 진입
- 광고 또는 패키지 구매 후 원래 액션 자동 재실행

## 10. 안전/운영 정책

### 10.1 사용자 안전
- 프로필 신고
- 채팅 신고
- 사용자 차단
- 차단 시 추천/채팅/노출 제외

### 10.2 운영 심사
- HIGH / LOW 리스크 큐 분리
- detail + report + matches highlight + audit log 제공
- 운영 액션:
  - APPROVE
  - REJECT
  - SUSPEND
  - UNSUSPEND
  - MASK_TOGGLE

### 10.3 운영 단축키
- `Space` 마스킹
- `Enter` 승인
- `Backspace` 반려
- `J/K` 하이라이트 이동
- `?` 도움말

## 11. 현재 정책상 가장 위험한 불일치

1. 광고 보상 문구 일부가 실제 지급량(+1)과 어긋남
2. Deep Read는 상위 명세상 중요하지만 실제 정책/UI 연결이 약함
3. 운영 정책은 강한데 사용자 안전 UX는 일부 약함
4. OB-04 직장 인증 정책은 문서상 존재하지만 현재 화면은 없음

## 13. 결론

GraceLink의 현재 정책은 단순한 소개 앱 정책이 아니다.

- 승인 게이트
- 텍스트 중심 탐색
- 제한된 행동 비용
- 운영 심사 레이어

이 네 가지가 함께 제품 성격을 만든다.

따라서 고도화 작업은 화면 하나만 바꾸는 식이 아니라, **정책과 문서까지 같이 움직이는 방식**으로 해야 한다.
