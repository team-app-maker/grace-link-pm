# L1 / L2 사용자 용어 치환 인벤토리

> SSOT 연계 문서: `docs/SSOT.md`

## 1. 문서 목적

이 문서는 GraceLink에서 내부 용어 `L1`, `L2`를 사용자 노출 용어로 바꿀 때 기준으로 쓰는 **치환 인벤토리 문서**다.

이 문서의 역할은 세 가지다.
- 최종 용어를 확정한다.
- 어떤 문구를 무엇으로 바꿀지 한 번에 본다.
- 실제 구현/문서/운영/QA까지 치환 범위를 빠뜨리지 않게 한다.

## 2. 최종 네이밍 결정

### 2.1 원칙
- 사용자에게는 `L1`, `L2`를 직접 보여주지 않는다.
- 사용자에게는 의미가 드러나는 문구만 보여준다.
- 두 레벨이 동시에 존재하는 경우에만 깊이 차이를 설명한다.

### 2.2 최종 권고안

| 내부 용어 | 사용자 노출 기본값 | 보조 설명 |
| --- | --- | --- |
| L1 | **나의 신앙 고백** | 짧고 먼저 읽는 핵심 고백 |
| L2 | **더 깊은 신앙 고백** | 더 길고 깊은 간증/서사 |

### 2.3 왜 이 안으로 가는가

#### `나의 신앙 고백`
- 사용자가 “이건 내 신앙을 말하는 영역이구나” 바로 이해한다.
- 자기소개보다 GraceLink의 정체성과 더 잘 맞는다.
- 질문/답변형 콘텐츠를 하나의 의미 있는 섹션으로 묶어준다.

#### `더 깊은 신앙 고백`
- `나의 신앙 고백`과의 연속성이 있다.
- L2가 더 긴 이야기라는 점이 자연스럽게 전달된다.
- 기존 `L2`처럼 기술적이지 않고, 기존 `깊은 신앙 이야기`보다 더 일관된 체계를 만든다.

## 3. 사용자에게 보여줄 때의 규칙

### 3.1 섹션 제목
- L1 섹션 제목: `나의 신앙 고백`
- L2 섹션 제목: `더 깊은 신앙 고백`

### 3.2 CTA / 버튼 / 라벨
- `L1 질문` → `나의 신앙 고백`
- `L1 답변 작성` → `나의 신앙 고백 작성`
- `L2 미션` → `더 깊은 신앙 고백 작성`
- `L1 전체 열람` → `나의 신앙 고백 전체 보기`
- `L2 심층 간증 열람` → `더 깊은 신앙 고백 보기`

### 3.3 내부 문서/코드 표기
- 내부 코드/타입/정책 문서는 `L1`, `L2` 유지 가능
- 다만 설명문에서는 병기 권장
  - `L1(나의 신앙 고백)`
  - `L2(더 깊은 신앙 고백)`

## 5. 단계별 적용 순서

### Phase 1 — 문서/카피 기준선
- SSOT 문서 반영
- PRD/IA/로드맵 반영
- 치환 인벤토리 확정

### Phase 2 — 사용자 UI 문자열 치환
- 온보딩
- 추천 empty state
- 프로필 상세
- 프로필 수정

### Phase 3 — 운영/QA 문서 반영
- API 설명문 병기
- QA 테스트 시나리오 반영
- 운영 설명문 병기

## 6. 절대 하지 말아야 할 것

- `L1`, `L2`를 UI에 그대로 남기는 것
- L1/L2를 둘 다 같은 라벨만 쓰고 차이를 완전히 숨기는 것
- 내부 모델명까지 한 번에 다 갈아엎어서 타입/정책 추적성을 깨는 것

## 7. 최종 결론

사용자에게는 앞으로 아래 체계를 공식 용어로 쓴다.

- **나의 신앙 고백**
- **더 깊은 신앙 고백**

이 두 용어가 GraceLink의 신앙 정체성을 가장 자연스럽게 드러내면서도, L1/L2의 레벨 차이를 유지하는 현재 최선의 안이다.
