# 홈 개편 와이어프레임 상세 명세

> SSOT 연계 문서: `docs/SSOT.md`

## 1. 문서 목적

이 문서는 홈 화면 개편의 **실행 가능한 와이어프레임 명세서**다.

목표는 아래 세 가지다.
- 현재 홈을 무엇으로 바꿀지 명확히 정한다.
- 디자이너/개발자가 같은 화면을 상상하게 만든다.
- 구현 티켓으로 바로 쪼갤 수 있는 수준까지 구체화한다.

## 2. 홈 개편 목표

현재 홈은 말씀/공지 중심이라 브랜드 톤은 좋지만, 사용자가 “지금 무엇을 해야 하는지”는 약하다.

개편 후 홈은 다음 역할을 해야 한다.
- 오늘의 상태를 요약한다.
- 가장 먼저 해야 할 행동을 하나만 크게 보여준다.
- 소개 / 인박스 / 채팅 / 보관함 / 스토어로 이어지는 허브가 된다.
- 말씀/공지는 남기되 1순위 행동을 가리지 않는다.

## 3. 홈 정보 구조

### 블록 순서
1. 상태 헤더
2. Primary Action Card
3. 3-way summary
4. 신뢰 / 준비 상태
5. 오늘의 말씀
6. 공지 / 운영 안내

## 5. 블록별 상세 명세

### 5.1 상태 헤더

#### 구성
- 타이틀: `홈`
- 서브타이틀: `오늘의 흐름`
- 상태 요약 텍스트
- 만나 잔액 pill

#### 데이터
- 오늘의 소개 남은 수
- 받은 신청 수
- 답장 필요 채팅 수
- 만나 잔액

#### 예시 카피
- `오늘의 소개 2명이 준비됐어요`
- `받은 신청 1건 · 답장이 필요한 대화 2개`

### 5.2 Primary Action Card

#### 목적
사용자가 앱을 열고 가장 먼저 해야 할 행동을 1개만 보여준다.

#### 우선순위 규칙
1. 받은 신청 응답 필요
2. 답장 필요한 채팅
3. 오늘의 소개 미열람
4. 보관함 만료 임박
5. 모두 완료

#### 카드 구성
- 소제목: `지금 가장 먼저 할 일`
- 본문: 상태 요약
- CTA 1개

#### 예시
- `받은 신청 1건이 18시간 뒤 만료돼요`
- 버튼: `인박스 보러가기`

### 5.3 3-way summary

#### 구성
- 카드 3개
  - 소개
  - 인박스
  - 채팅

#### 각 카드 요소
- 제목
- 현재 개수/상태
- CTA

#### 예시 카피
- `소개 / 아직 2명`
- `인박스 / 답변 대기 1건`
- `채팅 / 답장 필요한 대화 2개`

### 5.4 신뢰 / 준비 상태

#### 목적
사용자에게 프로필/신뢰 상태를 보여준다.

#### 포함 요소
- 교회 인증 여부
- 프로필 완성도
- 미작성 심화 항목

#### 예시 카피
- `교회 인증 완료`
- `프로필 완성도 85%`
- `더 깊은 신앙 고백이 아직 비어 있어요`

### 5.5 오늘의 말씀

#### 원칙
- 지금처럼 hero 영역으로 두지 않는다.
- compact card로 축소한다.
- 브랜드 톤은 유지하되 행동을 가리지 않게 한다.

#### 예시 구성
- 섹션 제목: `오늘의 말씀`
- 말씀 한 줄
- 펼치기 또는 자세히 보기 optional

### 5.6 공지 / 운영 안내

#### 원칙
- 홈 최하단으로 이동
- 카드 1~2개 수준으로 정리
- 중요도 높은 것만 badge 표시

## 6. 상태별 홈 화면

### Case A — 기본 상태
- 오늘의 소개 남아 있음
- 받은 신청 없음
- 답장 필요 없음
- Primary Action = 오늘의 소개 보러가기

### Case B — 응답 필요
- 받은 신청 1건 이상
- Primary Action = 인박스 보러가기

### Case C — 채팅 우선
- 답장 deadline 임박
- Primary Action = 채팅으로 이동

### Case D — 모두 완료
- 소개 없음, 응답 없음, 답장 없음
- Primary Action = 보관함 보기 또는 무료 만나/스토어 진입

## 7. 홈 카피 원칙

### 탭명
- 현재: `오늘`
- 권장: `홈`

### 헤더명
- 권장 1: `오늘의 흐름`
- 권장 2: `지금 해야 할 일`

### 금지 카피
- 너무 추상적인 문장만 있는 hero
- 행동을 설명하지 않는 상태 문구
- 내부 모델 용어(L1/L2) 노출

## 8. 디자인 원칙

- 상단은 정보, 중앙은 행동, 하단은 콘텐츠
- 숫자는 짧게, 행동은 크게
- CTA는 한 블록에 하나만 강하게
- 말씀/공지보다 상태/행동이 먼저
- 한 화면 안에 사용자의 하루 흐름이 보이게

## 9. 구현 시 필요한 데이터

- 남은 추천 수
- 받은 신청 수
- 답장 필요 채팅 수
- 만나 잔액
- 교회 인증 여부
- 프로필 완성도 계산값
- L2/심화 작성 여부
- 공지 데이터

## 10. 수용 기준

- 홈 진입 3초 내 사용자가 첫 행동을 이해할 수 있다.
- 홈에서 소개/인박스/채팅/보관함/스토어 상태를 한 번에 확인할 수 있다.
- 말씀/공지는 유지되지만 메인 액션을 가리지 않는다.
- 내 정보 화면으로 가지 않아도 핵심 상태를 파악할 수 있다.
