# 메인 화면 개편 + L1/L2 용어 전략

> SSOT 연계 문서: `docs/SSOT.md`

## 1. 왜 이 문서가 필요한가

현재 GraceLink의 가장 큰 제품 문제는 두 가지다.

1. **메인 화면(현재 홈)이 너무 수동적이다.**
   - 말씀/공지 중심이라 브랜드 톤은 보이지만, 사용자가 “지금 무엇을 해야 하는지”는 약하다.
   - 반대로 핵심 행동(오늘의 소개, 인박스, 채팅, 보관함)은 여기저기 흩어져 있다.

2. **L1 / L2 용어가 사용자 언어가 아니다.**
   - 팀 내부 코드/정책 용어로는 유효하다.
   - 하지만 사용자에게는 기능 의미가 직관적으로 전달되지 않는다.
   - 특히 고도화 단계에서 이 용어를 계속 노출하면 학습비용과 거리감이 생긴다.

이 문서는 위 두 문제를 해결하기 위한 **제품/UX/카피 관점 제안서**다.

---

## 2. 현재 문제 정의

### 2.1 메인 화면 문제

현재 홈은 다음 특성을 가진다.
- 차분한 톤의 말씀/공지 허브
- 제품 철학은 보이지만 행동 유도는 약함
- 추천/인박스/보관함/채팅으로 이어지는 운영 중심 흐름과 분리됨

즉, 지금 홈은 “브랜드는 말하지만, 제품을 움직이지는 않는 화면”에 가깝다.

### 2.2 L1/L2 용어 문제

현재 용어는 내부적으로는 편하지만, 사용자 입장에서는 아래 문제가 있다.
- 무엇이 짧고 무엇이 깊은지 바로 알 수 없다.
- 신앙 문답인지, 자기소개인지, 해금 콘텐츠인지 구분이 어렵다.
- 온보딩/프로필/해금/운영 문서에서 같은 용어가 서로 다른 느낌으로 보인다.

즉, `L1`, `L2`는 내부 모델명으로는 괜찮지만 **사용자 언어로는 실패한 라벨**이다.

## 3. 메인 화면 개편 방향

## 3.1 목표

메인 화면은 앞으로 아래 역할을 해야 한다.

- 사용자가 앱을 열었을 때 **오늘 무엇을 해야 하는지** 바로 알게 한다.
- 현재 상태(심사, 소개, 응답, 대화, 보관)를 한눈에 보여준다.
- 행동 우선순위를 정리해 준다.
- 말씀/공지/브랜드 톤은 유지하되, 메인 액션을 가리지 않는다.

## 3.2 추천 포지션

### 추천안: 홈 = “오늘의 허브”, 소개 = “탐색 화면” 유지

즉,
- 홈은 행동 우선순위를 정리하는 **컨트롤 타워**
- 오늘의 소개 탭은 몰입해서 읽는 **탐색 화면**

이 구조가 가장 안전하다.

#### 이유
- 현재 `오늘의 소개` 화면은 이미 카드 중심 구조가 살아 있다.
- 이 화면을 홈에 억지로 합치면 철학도 흐리고 탐색도 흐린다.
- 대신 홈에서 `오늘의 소개`로 강하게 연결해주면 된다.

## 4. 메인 화면 정보 구조 제안

## 4.1 추천 레이아웃

### A. 상단: 오늘 상태 헤더
- 인사 + 닉네임
- 오늘 남은 소개 수
- 만나 잔액
- 프로필/신뢰 상태 배지

예시:
- `오늘의 소개 2명 남았어요`
- `받은 신청 1건`
- `답장이 필요한 대화 2개`

### B. 1순위 행동 카드
가장 먼저 해야 할 행동 1개만 크게 보여준다.

우선순위 예시:
1. 받은 신청 응답
2. 답장 필요한 채팅
3. 오늘의 소개 아직 안 봄
4. 만료 임박 보관함
5. 모두 처리 완료

이 카드 하나만 잘 만들어도 홈의 목적성이 살아난다.

### C. 진행 현황 3분할
- 오늘의 소개
- 인박스
- 채팅

각 카드에 아래만 보여준다.
- 개수
- 상태
- 바로가기 CTA

예시:
- `오늘의 소개 / 아직 2명`
- `인박스 / 답변 대기 1건`
- `채팅 / 24시간 내 답장 2건`

### D. 신뢰 / 준비 상태 카드
- 교회 인증
- 프로필 완성도
- 아직 작성 안 한 심화 항목(L2 등 향후)

### E. 말씀 / 공지 / 운영 공지
현재 홈의 말씀/공지 영역은 없애지 말고 **아래로 내린다**.

권장 방식:
- 말씀은 hero가 아니라 compact card
- 공지는 collapse 가능한 리스트
- 운영 공지는 badge + 1줄 요약

즉, “좋은 톤”은 유지하되 “첫 행동”을 가리지 않게 한다.

## 6. 홈 화면에서 바꿔야 할 핵심 카피

### 현재 문제
- “오늘”은 의미가 넓고 행동이 약하다.
- 말씀/공지 중심이라 제품 목적보다 콘텐츠 허브처럼 느껴진다.

### 추천
탭명은 아래 둘 중 하나를 추천한다.

#### 1안 — 유지형
- 탭명: `오늘`
- 홈 제목: `오늘의 진행 상황`

장점:
- 현재 구조와 충돌 적음
- 과한 리브랜딩 없이 개선 가능

#### 2안 — 명확형 (추천)
- 탭명: `홈`
- 홈 제목: `오늘의 흐름`

장점:
- 허브 화면이라는 의미가 더 분명함
- `오늘의 소개` 탭과 역할 충돌이 줄어듦

### 추천 결론
- **탭명은 `홈`으로 변경**
- 화면 제목은 `오늘의 흐름` 또는 `지금 해야 할 일`

## 7. L1 / L2 사용자 언어 전략

## 7.1 원칙

사용자에게는 내부 모델명을 보여주지 않는다.

- 내부 코드/정책: `L1`, `L2` 유지 가능
- 사용자 UI/문서/카피: 의미 기반 용어로 전환

## 7.2 추천 네이밍 안

### 최종 추천안
| 내부 용어 | 사용자 노출 용어 | 역할 |
| --- | --- | --- |
| L1 | **나의 신앙 고백** | 먼저 읽는 핵심 고백 |
| L2 | **더 깊은 신앙 고백** | 더 깊은 간증/신앙 서사 |

### 왜 이 안이 좋은가

#### `나의 신앙 고백`
- 사용자가 신앙 영역이라는 것을 즉시 이해한다.
- GraceLink의 정체성과 가장 직접적으로 맞닿아 있다.
- 질문/답변형 콘텐츠를 하나의 의미 있는 자기 고백 섹션으로 묶어준다.

#### `더 깊은 신앙 고백`
- `나의 신앙 고백`과 계열이 같다.
- L2가 심화 영역이라는 점을 자연스럽게 전달한다.
- 내부 레벨 구조를 숨기면서도 깊이 차이는 유지한다.

## 8. 대체안 비교

### A안
- L1 = 나의 신앙 고백
- L2 = 더 깊은 신앙 고백

**평가:** 가장 균형 좋음 → **추천**

### B안
- L1 = 신앙 고백
- L2 = 더 깊은 신앙 고백

장점:
- 부드럽고 부담이 적다

단점:
- L1 단독 라벨이 조금 일반적이다

### C안
- L1 = 나의 신앙 고백
- L2 = 신앙 고백 더 보기

장점:
- 의미는 명확함

단점:
- CTA에는 쓸 수 있지만 섹션 제목으론 다소 어색하다

### D안
- L1 = 짧은 신앙 고백
- L2 = 긴 신앙 고백

장점:
- 이해는 쉬움

단점:
- 매력도와 브랜드 톤이 약함
- 길이만 강조돼 내용 가치가 약해짐

## 9. 실제 적용 카피 제안

## 9.1 온보딩
- `L1 질문` → `나의 신앙 고백`
- `L1 답변 작성` → `나의 신앙 고백 작성`
- `L2 미션` → `더 깊은 신앙 고백 작성`

## 9.2 프로필 상세
- `Conversation Starters` → `더 깊은 신앙 고백`
- `My Philosophy` → `나의 신앙 고백`
- `L2 심층 간증` → `더 깊은 신앙 고백`

## 9.3 해금/과금
- `L1 전체 열람` → `나의 신앙 고백 전체 보기`
- `L2 심층 간증 열람` → `더 깊은 신앙 고백 보기`

## 9.4 운영/문서
- 운영자용 문서에서도 가능하면 설명문에는 사용자 용어 병기
  - 예: `L1(나의 신앙 고백)`
  - 예: `L2(더 깊은 신앙 고백)`

- 이유: 깊은 설정 구조를 레벨별로 어떻게 나누는지 참고하기 좋다.

### 12) Material Design — Settings pattern
- 링크: https://m1.material.io/patterns/settings.html
- 참고 문장: 라벨만으로 충분하면 보조 문장을 반복하지 말라고 안내한다.
- 우리에게 주는 시사점: `L1`, `L2`처럼 설명이 필요한 애매한 용어보다, 처음부터 의미가 드러나는 라벨이 낫다.

### 13) Hennepin County UI copy guide
- 링크: https://www.hennepincounty.gov/design-system/content/writing-guide/digital-text-standards
- 참고 문장: UI copy는 짧고 명확해야 하며, `OK` 같은 모호한 단어를 피하고 일관된 action word를 써야 한다.
- 우리에게 주는 시사점: `L1`, `L2`는 action이나 object가 아니므로 UI 레이블로 부적합하다.

### 14) UW UX Labels guide
- 링크: https://uxdesign.uw.edu/interaction/labels.html
- 참고 문장: 라벨은 짧고 구체적이어야 하며, 비슷한 요소를 구분할 수 있어야 한다.
- 우리에게 주는 시사점: `나의 신앙 고백`과 `더 깊은 신앙 고백`은 같은 계열 안에서 구분이 된다.

### 15) N Brown UX writing guide
- 링크: https://ux.nbrown.co.uk/content/how-we-write
- 참고 문장: jargon을 피하고, 표현을 일관되게 쓰고, progressive disclosure로 필요한 정보만 단계적으로 보여주라고 말한다.
- 우리에게 주는 시사점: 홈과 온보딩 모두 “한 번에 다 보여주기”보다 단계적 공개가 맞다.

## 11. 최종 권고안

### 메인 화면
- 현재 홈을 “오늘의 허브”로 개편한다.
- 말씀/공지는 내려 보내고, 지금 해야 할 행동을 위로 올린다.
- `오늘의 소개` 탭은 유지하되, 홈에서 강하게 연결한다.
- 탭명은 `홈`으로, 화면 제목은 `오늘의 흐름` 계열로 바꾼다.

### L1/L2 용어
- 내부 코드/정책에서는 유지 가능
- 사용자 노출 용어는 아래로 고정 권장
  - **L1 → 나의 신앙 고백**
  - **L2 → 더 깊은 신앙 고백**

### 문서/운영 원칙
- 앞으로 제품/UI/운영 문서에서는 사용자 용어를 우선 사용한다.
- 내부 기술 문서에서만 병기 형태로 `L1(나의 신앙 고백)`처럼 사용한다.
