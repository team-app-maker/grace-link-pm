# 구현 티켓 / 작업 순서 분해

> SSOT 연계 문서: `docs/SSOT.md`

## 1. 문서 목적

이 문서는 현재 SSOT 문서를 실제 구현 작업으로 옮기기 위한 **실행 티켓 분해 문서**다.

기획/디자인/개발/QA가 공통으로 볼 수 있도록,
- 작업 단위
- 의존 순서
- 완료 기준
- 검증 포인트
를 정리한다.

## 2. 에픽 구조

### Epic A. 용어 전환
- 목표: 사용자 노출에서 `L1`, `L2` 제거

### Epic B. 홈 개편
- 목표: 홈을 말씀/공지 허브에서 행동 허브로 전환

### Epic C. 구조 정리
- 목표: 인박스/보관함/스토어 발견성 개선

## 3. 티켓 목록

## Epic A — 용어 전환

### A-1. 치환 대상 정리
- 목적: UI/문서/QA/운영에서 `L1`, `L2`가 어디 쓰이는지 확정
- 산출물: 치환 체크리스트
- 기준 문서: `docs/ssot/l1-l2-replacement-inventory.md`
- 완료 기준: 사용자 노출 문자열 목록 확정

### A-2. 온보딩 카피 치환
- 대상: `ob-07`, `ob-08`
- 변경 방향:
  - `L1 질문` → `나의 신앙 고백`
  - 관련 버튼/설명문 치환
- 완료 기준: 온보딩에서 `L1` 노출 제거

### A-3. 프로필 상세 카피 치환
- 대상: `profile/[targetUuid]`
- 변경 방향:
  - `My Philosophy` → `나의 신앙 고백`
  - `Conversation Starters` → `더 깊은 신앙 고백` 또는 하위 보조 카피
- 완료 기준: 프로필 상세에서 내부 용어 제거

### A-4. 추천/미션 카피 치환
- 대상: `reco.tsx`
- 변경 방향:
  - `L2 미션 완료` → `더 깊은 신앙 고백 작성`
- 완료 기준: 홈/추천/empty state에서 내부 용어 제거

### A-5. 문서/QA/운영 병기 정리
- 대상: `docs/*`
- 방향: 내부 모델명은 병기만 허용
- 완료 기준: 사용자 문서에서 최종 용어 체계 통일

## Epic B — 홈 개편

### B-1. 홈 데이터 요구사항 정리
- 필요한 상태값 정리
  - 남은 소개 수
  - 받은 신청 수
  - 답장 필요 채팅 수
  - 만나 잔액
  - 인증/완성도 상태
- 완료 기준: UI 블록별 데이터 매핑 완료

### B-2. 홈 정보 구조 개편
- 대상: `app/(tabs)/index.tsx`
- 변경 방향:
  - 상단 상태 헤더
  - primary action card
  - 3-way summary
  - 신뢰/준비 상태
  - 말씀/공지 하단 배치
- 완료 기준: 와이어프레임 구조 반영

### B-3. 홈 카피 적용
- 탭명: `홈`
- 헤더명: `오늘의 흐름`
- CTA 카피: 행동 중심으로 재작성
- 완료 기준: 홈의 모든 핵심 텍스트 교체

### B-4. 홈 디자인 polish
- 카드 계층 / 간격 / hierarchy 정리
- 정보-행동-콘텐츠 레벨 구분 강화
- 완료 기준: 런타임 캡처 재생성 가능한 수준

### B-5. 홈 캡처 및 문서 갱신
- 대상 문서
  - `docs/runtime-captures/ios/README.md`
  - `docs/ux-screen-inventory.md`
  - `docs/prd-reverse-engineered-current-app.md`
- 완료 기준: 홈 캡처와 설명이 새 구조 반영

## Epic C — 구조 정리

### C-1. 인박스 발견성 개선
- 후보
  - 탭 승격
  - 홈 요약 카드 직행
- 완료 기준: 사용자 첫 경로 명확화

### C-2. 보관함 발견성 개선
- 후보
  - 홈 요약 카드
  - 내 정보 보조 유지
- 완료 기준: 보관함이 숨은 기능이 아니게 됨

### C-3. 스토어 구조 정리
- 대상
  - `/shop`
  - `/(tabs)/shop`
- 완료 기준: 단일 진입 구조 결정

## 4. 권장 구현 순서

1. A-1 치환 대상 정리
2. B-1 홈 데이터 요구사항 정리
3. B-2 홈 구조 개편
4. B-3 홈 카피 적용
5. A-2 / A-3 / A-4 UI 용어 전환
6. C-1 / C-2 / C-3 구조 정리
7. B-4 디자인 polish
8. A-5 문서/QA 정리
9. B-5 캡처/문서 갱신

## 5. 각 티켓 공통 완료 기준

- 관련 화면 캡처 갱신 가능
- 관련 SSOT 문서 갱신
- lint / typecheck / test 통과
- 기존 정책과 충돌 여부 확인

## 6. QA 묶음

### QA-1 용어 전환 QA
- 사용자 화면에서 `L1`, `L2`가 남아 있는지 확인
- 온보딩 / 프로필 / 추천 / 편집 화면 확인

### QA-2 홈 개편 QA
- 첫 행동이 3초 내 인지 가능한지
- 상태 요약이 실제 데이터와 맞는지
- CTA가 올바른 라우트로 이동하는지

### QA-3 구조 정리 QA
- 인박스/보관함/스토어 진입이 쉬워졌는지
- 기존 핵심 기능 누락 없는지

## 7. 바로 생성 가능한 구현 이슈 제목 예시

- `홈을 행동 허브 구조로 개편`
- `온보딩 L1 용어를 나의 신앙 고백으로 치환`
- `프로필 상세 L2 용어를 더 깊은 신앙 고백으로 치환`
- `인박스/보관함 발견성 개선`
- `스토어 진입 구조 단일화`
- `홈 개편 후 런타임 캡처 및 SSOT 문서 갱신`
