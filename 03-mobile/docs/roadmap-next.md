# GraceLink 다음 고도화 로드맵

> SSOT 연계 문서: `docs/SSOT.md`

## 1. 목적

이 문서는 현재 제품 기준선 위에서, 무엇을 어떤 순서로 고도화해야 하는지 정리한 **실행 우선순위 문서**다.

## 2. 최우선 과제

### P1. 메인 화면 개편
- 홈을 말씀/공지 허브에서 행동 허브로 전환
- 사용자가 앱을 열었을 때 “무엇을 먼저 해야 하는지” 보이게 만들기
- 관련 문서: `docs/ssot/home-redesign-and-copy-strategy.md`

### P2. L1/L2 사용자 용어 전환
- 사용자 노출 용어에서 `L1`, `L2` 제거
- 권장 치환:
  - `L1` → `나의 신앙 고백`
  - `L2` → `더 깊은 신앙 고백`
- 적용 대상:
  - 온보딩
  - 프로필 상세
  - 해금/과금
  - 운영 문서
  - QA 문서

### P3. 숨김 핵심 기능 발견성 개선
- 인박스
- 보관함
- 스토어
- 홈 개편과 함께 접근성을 재정리

## 3. 다음 단계 과제

### P4. 온보딩 완성도 보강
- 약관 UI
- OB-04 직장 인증 여부 결정
- 교회/회사 인증과 필터 연결

### P5. BM/콘텐츠 차별화 복구
- L2 실제 작성 플로우 연결
- Deep Read 정책 적용
- 프로필 공개 레벨 재설계

### P6. 안전/운영 연결 강화
- 사용자 신고/차단과 운영 큐 정합화
- 운영 액션 권한 모델 정리

## 4. 제안 순서

1. 메인 화면 개편
2. L1/L2 용어 전환
3. 인박스/보관함/스토어 발견성 개선
4. 온보딩 완성도 보강
5. Deep Read / L2 복구
6. 운영 고도화

## 5. 성공 기준

- 홈 진입 후 사용자가 다음 행동을 3초 안에 이해할 수 있다.
- 사용자 문서와 UI에서 `L1`, `L2` 노출이 사라진다.
- 추천/인박스/보관함/채팅이 하나의 행동 흐름으로 연결된다.
- 정책/카피/문서 불일치가 줄어든다.
