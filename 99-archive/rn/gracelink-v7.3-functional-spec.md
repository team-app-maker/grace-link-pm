# 1) 제품 원칙 및 범위

## 1.1 Iron Rules
1) **No Photo**  
- 앱 내 **프로필 사진 업로드/열람/전송 기능 없음**  
- 이미지 첨부/전송 UI 없음(채팅 포함)

2) **신뢰가 상품**  
- PASS 본인인증(19+) + 중복가입 방지(CI 해시)  
- 운영자 **수동 승인**(Approve/Reject/Needs Edit/Suspend)

3) **희소성(High-Intent)**  
- 추천은 **하루 2명 기본**  
- 추가 추천은 유료(+2) **하루 2회까지만**  
- 무제한/리프레시/재요청(무한 뽑기) 불가

4) **착한 환급(신뢰 정책)**  
- **거절(REJECT)해도 신청자 100% 환급**  
- **48시간 무응답(EXPIRE)도 신청자 100% 환급**  
- 단, 무응답/읽씹은 **노출 페널티(비가시 지표)**로 생태계 방어

5) **개인정보/외부유도는 수동 대응 중심**  
- AI/완전 자동 차단 없음  
- 대신 운영 큐 분리(LOW/HIGH RISK) + 하이라이트로 심사 효율화

## 1.2 In Scope (필수 구현)
- PASS 인증, 교단/지역/교회 등록(검색), 직장 이메일 인증(선택)
- 연락처 차단, 같은 교회/회사 제외 필터
- L1 Q&A(필수) + L2 심층 간증(미션) + Anti-Paste
- 좋아하는 말씀 구절/찬양(CCM) 입력
- 추천 카드 48h LOCK + 7d 삭제 + 유료 해제
- Deep Read(텍스트 잠금) 과금(동일 상대 1회 결제 후 재열람 무료)
- NORMAL/SUPER 매칭, 48h 응답, 환급, 채팅 텍스트
- 단방향 잠수(Asymmetric) 페널티/보상
- 결제 UX 스무딩(결제 후 원래 액션 자동 완료)
- **보상형 광고(Rewarded Ads)로 ‘만나’ 지급**
- 운영 어드민: 큐/하이라이트/Hotkey/감사 로그

## 1.3 Out of Scope
- 사진 기반 매칭/사진 열람 BM 일체
- 주말 배려 타이머(Weekend Grace Timer) **삭제**
- 음성/영상/파일 전송
- 무제한 추천/무제한 신청

# 2) IA/메뉴 구조 및 핵심 플로우

## 2.1 탭 구조(4탭)
- **HOME**: 오늘의 추천(2) / +2 확장 / 보관함(VAULT) / EMPTY 리텐션
- **INBOX**: 받은 신청 / 보낸 신청(상태/타이머)
- **CHAT**: 채팅 리스트/채팅방(텍스트만)
- **MY**: 내 프로필/인증/필터/상점/무료만나(광고)/설정

## 2.2 핵심 플로우
1) OB → SUBMITTED → 운영 승인(APPROVED)  
2) HOME에서 카드 확인 → (필요 시) **Deep Read 결제**  
3) 마음에 들면 매칭 신청(NORMAL/SUPER)  
4) INBOX에서 수락/거절(48h) → ACCEPT 시 채팅 생성  
5) 채팅에서 첫 인사/답장 타이머 + 잠수 정책  
6) 잔액 부족 시 페이월(구매 or 광고) → 결제/광고 완료 즉시 액션 자동 진행

# 3) 경제 시스템: 만나 + 보상형 광고

## 3.1 재화 정의
### 3.1.1 만나(Manna)
- 앱 내 유료 재화(결제) + 무료 재화(광고)로 획득
- **단일 통화**로 UX 단순화(“GP” 제거)

### 3.1.2 잔액 분리(권장: 회계/어뷰징 방어)
- `paid_manna_balance`: 결제 획득
- `bonus_manna_balance`: 광고/보상 획득
- 소비 우선순위: **bonus → paid** (유저 체감 ‘무료’ 최대화)

## 3.2 아이템/액션 가격표(기본안)
> 가격/패키지는 Remote Config로 조정 가능하게 설계

### 3.2.1 Deep Read (텍스트 잠금 해제)
| 액션 | 비용(만나) | 설명 | 재결제 |
|---|---:|---|---|
| L1 전체 열람 | 1 | L1 답변 3개를 전체로 펼쳐보기 | 동일 상대 1회(영구) |
| L2 심층 간증 열람 | 2 | L2 6문항 전체 열람 | 동일 상대 1회(영구) |
| 말씀/찬양 “이유” 열람 | 1 | 좋아하는 말씀/찬양의 이유/간증 열람 | 동일 상대 1회(영구) |

### 3.2.2 추천/보관/시간(Time) 과금
| 액션 | 비용(만나) | 제약 |
|---|---:|---|
| +2 추천 확장 | 1 | 하루 2회(총 +4) |
| LOCKED 카드 열기 | 1 | 카드 생성 48h 경과 |
| 보관 연장(+3일) | 1 | 프로필당 최대 2회 |
| 삭제 카드 부활(Revive) | 2 | 주 2회 제한 권장 |

### 3.2.3 매칭(행동) 과금
| 액션 | 비용(만나) | 설명 |
|---|---:|---|
| NORMAL 매칭 신청 | 3 | 신청자 차감 |
| NORMAL 매칭 수락 | 3 | 수락자도 차감(쌍방 과금) |
| SUPER 매칭 신청 | 6 | 수락자 무료 + 받은함 상단 고정(pinned) + 강조 |
| SUPER 매칭 수락 | 0 | 무료 수락 |

## 3.3 보상형 광고(Rewarded Ads) — GP 대체 핵심
### 3.3.1 목표
- “유료 결제”를 못/안 하는 유저에게도 **행동을 이어갈 최소 연료(만나)** 제공
- 단, **속도/상한**으로 구매 대비 효율을 낮춰 ARPU를 방어

### 3.3.2 기본 룰(권장값)
- 1회 광고 완료 → **보너스 만나 +1**
- 일일 상한: **3회/일** (총 +3 보너스 만나)
- 쿨다운: 10분(또는 1시간) 중 택1
- 자격: `APPROVED` 유저만 가능(가입 미승인/정지 계정 제외)
- 지급 원칙: 서버 검증 후 지급(클라 신뢰 금지)

> 운영에서 `daily_cap`, `cooldown`, `reward_amount`는 Remote Config로 조절

### 3.3.3 광고 노출 지점(필수)
1) **PAYWALL_BOTTOM_SHEET**(잔액 부족 시)  
- “패키지 구매”와 함께 “광고 보고 만나 1개 받기” 옵션 제공  
- 광고 시청 후 **즉시 원래 액션 자동 진행**(Intent-preserving)

2) **MY > 무료 만나**  
- “오늘 남은 무료 만나: 2/3” 표시 + 광고 보기 버튼

3) **EMPTY(추천 0장)**  
- “광고 보고 +2 추천 비용(만나 1) 만들기” CTA(선택)

### 3.3.4 어뷰징/부정 방어(필수)
- 광고 완료 이벤트는 `reward_token`을 서버로 전송 → 서버가 네트워크(Ad SDK)의 검증 API/서명 확인 후 지급
- Rate limit:
  - 계정/디바이스/CI 기준 분당 요청 제한
  - 동일 토큰 재사용 방지(1회성)
- 제재:
  - 비정상 패턴 감지 시 광고 보상 기능 일시 중단(soft suspend) + 운영 큐

### 3.3.5 원장(ledger) 기록
- `transactions`에 반드시 기록:
  - currency: MANNA_BONUS
  - action_type: REWARD_AD
  - amount: +1
  - meta: ad_network, placement, token_hash

# 4) 온보딩/프로필 + 수동 심사

## 4.1 온보딩 단계(권장 Screen ID)
### OB-01 PASS 인증 & 약관
- PASS API 연동
- 만 19세 미만 차단
- CI 해시 저장(중복가입 방지)

### OB-02 기본 정보
- 닉네임(중복 허용 정책 택1), 성별, 출생년도, 교단, 지역(시/도 → **구/군**)

### OB-03 교회 등록(검색 기반)
- 교회 검색/선택(자기신고)
- “교회 추가 요청” 폼(선택)
- 추후 인증 완료 시 “교회 인증 배지”

### OB-04 직장 인증(선택)
- 회사 이메일 인증(메일 링크 확인)
- 인증 완료 시 배지 + “같은 회사 제외” 필터 활성화 가능

### OB-05 태그 선택(5~15)
- 카테고리: 성격/신앙가치/교회생활/연애관/라이프(부록 참고)
- 최소 5개 미만 Next 비활성

### OB-06 좋아하는 말씀/찬양(CCM) 입력 (**신규 필수**)
#### 1) 말씀(필수 1개 + 선택 2개)
- 입력: 성경책 + 장 + 절 (예: 요한복음 3:16)
- (선택) 이유/간증 80~200자

#### 2) 찬양(CCM) (선택 0~3개)
- 입력: 곡명(검색 가능 시) + 직접 입력 fallback
- (선택) 이유 40~120자

> 저작권 방어: 앱 UI 기본 노출은 “레퍼런스/곡명” 중심. 가사/전문 노출 금지.

### OB-07 L1 신앙 Q&A(필수)
- 6문항 중 3개 선택
- 각 답변 **150자 이상**
- Anti-Paste 적용(붙여넣기 차단 + 델타 입력 감지)

### OB-08 제출
- SUBMITTED → UNDER_REVIEW
- 심사 완료까지 HOME 제한(대기 화면 제공)

## 4.2 프로필 레벨(Progressive Profiling)
### L0(Preview, 무료)
- 기본 정보, 교회/직장 배지, 태그
- L1 답변: 각 1~2줄 미리보기
- 말씀 1개/찬양 1개: 레퍼런스/곡명만

### L1(필수, 유료 열람 가능)
- L1 전체(만나 1)로 상대의 답변 전체 열람

### L2(심층, 미션 + 유료 열람)
- 내 L2 작성은 “추천 품질”에 반영(비가시 지표)
- 상대 L2 열람은 만나 2(동일 상대 1회)

> 선택 옵션(악랄 BM, Remote Config):
> - “내 L2 1개 이상 작성 시 L2 열람 가능” (미작성자는 열람 불가)  
> - 또는 “미작성자는 L2 열람 비용 +1”(2→3)  
> → 초기 런칭은 **비용 +1**이 운영 부담이 적고 매출에 유리

## 4.3 수동 심사(운영)
- 상태: SUBMITTED → UNDER_REVIEW → APPROVED/REJECTED/NEEDS_EDIT/SUSPENDED
- 반려 시 “수정 요청” 가능(NEEDS_EDIT)
- 승인 후 HOME 오픈

# 5) 추천/카드 정책(7.2 기준)

## 5.1 추천 생성
- 매일 00:00 KST, 유저별 추천 2장 생성(기본)
- 추가 추천: +2(만나 1) 하루 2회

## 5.2 카드 수명/잠금/삭제
- 카드 생성 후 **48시간 내 상세 미열람** → LOCKED
- LOCKED 열기: 만나 1
- 카드 생성 후 **7일** → DELETED(완전 삭제)
- PASS 시 즉시 PASSED, 재노출 없음

## 5.3 보관함(VAULT)
- 최근 7일 카드 리스트
- 상태 배지: LOCKED/SEEN/D-n 삭제
- LOCKED는 보관함에서 열 때도 동일 비용(만나 1)
- 보관 연장(옵션): +3일(만나 1, 프로필당 최대 2회)

## 5.4 EMPTY(추천 0장) 화면
- 1차 CTA: “L2 질문 1개만 더 기록하기(3분)”
- 2차 CTA: “무료 만나 받기(광고)”
- 3차 CTA: “내일 추천 알림 켜기”

# 6) Deep Read(텍스트 잠금) 상세 기능

## 6.1 잠금 구조(권장 UI)
- L1: 질문 타이틀 + 답변 2줄 + “🔒 전체 보기(만나 1)”
- L2: 질문 리스트만 노출 + “🔒 심층 간증 열람(만나 2)”
- 말씀/찬양 이유: “🔒 이유 보기(만나 1)”

## 6.2 1회 결제 후 영구 열람 정책
- viewer-target 단위로 unlock 저장
- 한번 unlock하면:
  - 동일 상대가 다시 추천에 떠도 **재결제 없음**
  - INBOX/CHAT에서도 동일 상대의 잠금 영역은 열림
- 단, 카드가 7일 삭제되어도 unlock은 남는다(권장)
  - *정책 선택*: “계정 기간 영구” vs “30일” → 영구 권장(신뢰)

## 6.3 Deep Read 구매 시 넛지(카피)
- “한 번 열면 다시 결제하지 않아요.”
- “겉모습보다, 삶의 고백을 읽어보세요.”

# 7) 매칭 정책(7.2 + 7.3 하드닝)

## 7.1 매칭 신청
### MATCH-REQUEST Bottom Sheet
- NORMAL(3): “상대도 수락 시 3이 필요해요(쌍방)”
- SUPER(6): “상대는 무료로 수락 + 상단 고정(확률 UP)”

## 7.2 응답/환급(48h)
- 응답 기한: 48시간
- **거절도 환급**: 신청자 100% 환급
- 무응답 만료도 환급: 신청자 100% 환급
- 수락자 페널티(생태계 방어): **Seen/Unseen 차등**(비가시 노출 지표)

### Match Status
- PENDING
- ACCEPTED
- REJECTED_REFUNDED
- EXPIRED_REFUNDED_UNSEEN
- EXPIRED_REFUNDED_SEEN

### 페널티(예시, Remote Config)
- UNSEEN 만료: exposure_weight -10% for 3 days
- SEEN 만료: exposure_weight -30% for 7 days

## 7.3 “착한 방치” 방지 UX(필수)
- 거절 버튼 옆에 명확히 표기: “거절해도 신청자는 환급됩니다”
- 44h 리마인드 푸시(옵션): “응답 기한이 곧 끝나요. 수락/거절로 마음을 전해주세요.”

# 8) 채팅/단방향 잠수(Asymmetric) — *주말 연장 없음*

## 8.1 채팅 기본
- ACCEPTED 시 채팅방 생성
- 텍스트만(이미지/파일 버튼 없음)
- 시스템 메시지:
  - “서로를 존중하는 대화를 부탁드려요 🙏”

## 8.2 타이머 정책(기본 24h)
> 주말 배려 타이머 기능은 **삭제**한다.

- 첫 메시지 기한: 24h
- 답장 기한(상대 첫 답장): 24h

## 8.3 meaningful message(한국어 최적화)
- 메시지 전송 자체는 막지 않음
- 단, “의미 있는 첫 메시지”로 판정되어야 타이머/페널티 트리거

권장 판정:
- len_trimmed >= 5
- 한글/영문 포함(/[가-힣A-Za-z]/)
- 자음/모음 반복(ㅋㅋㅋ/ㅎㅎㅎ/ㅠㅠㅠ)만으로 구성 금지

## 8.4 단방향 잠수 처리(Asymmetric)
- A가 meaningful 메시지로 선톡 → B가 24h 내 reply 없음:
  - B 페널티(Seen/Unseen 차등 가능)
  - A 위로 보상: **보너스 만나 +1** (주 3회 상한 권장)
  - 채팅방은 “대화 종료” 상태로 전환(선택)

> 어뷰징 방어: 보상 지급은 “서버 판정” + 상한 + 동일 상대 1회 제한

# 9) 결제 UX 스무딩 + 환불/원장(ledger)

## 9.1 Intent-preserving Paywall(필수)
- 잔액 부족 시 상점 화면 이동 금지(가능하면)
- 현재 화면 위 `PAYWALL_BOTTOM_SHEET`에서:
  - 패키지 구매
  - 또는 광고 보고 보너스 만나 받기(가능할 때)

## 9.2 결제/광고 완료 후 자동 액션
- 결제/광고 완료 → 잔액 반영 → 원래 하려던 액션 자동 실행
- 실패 시 안내 + 재시도

## 9.3 원장(ledger) 무결성
- 모든 만나 변동은 transactions에 기록
- 환급은 REFUND 타입으로 기록
- 광고 보상은 REWARD_AD 타입으로 기록
- 동시성: match_request/accept/reject/refund는 사용자 row lock + 트랜잭션

# 부록 A) L1/L2 질문(예시 템플릿)
> 실제 문항은 운영에서 조정 가능. 핵심은 “간증급 질문”을 유지하되 가입 이탈을 막기 위해 **L1은 3문항 선택형**.

## L1(6개 중 3개 선택, 각 150자+)
- L1-01: 예수님을 인격적으로 만났던 순간이 있다면, 그때 어떤 변화가 있었나요?
- L1-02: 신앙이 흔들렸던 시기를 어떻게 지나왔나요?
- L1-03: 연애/결혼에서 ‘신앙’이 구체적으로 어떤 기준이 되나요?
- L1-04: 교회 공동체에서 어떤 모습으로 섬기고 있나요?
- L1-05: 당신이 중요하게 여기는 성품/가치(정직/절제/온유 등)는 무엇인가요?
- L1-06: 갈등이 생길 때, 기도/대화/회복을 어떤 방식으로 해왔나요?

## L2(심층 6문항, 각 150자+ 권장 300자)
- L2-01: 하나님 앞에서 가장 솔직했던 기도는 어떤 내용이었나요?
- L2-02: 가족/상처/관계에서 회복을 경험한 이야기
- L2-03: 사역/헌신/소명에 대한 생각
- L2-04: 경제/직업/삶의 방향에서 신앙이 준 선택
- L2-05: 연애에서 경계선(Boundary)을 어떻게 지키고 싶나요?
- L2-06: 결혼 후 ‘가정 예배/교회 생활’을 어떻게 그리나요?
